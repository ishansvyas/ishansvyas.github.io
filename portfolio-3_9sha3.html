SHA-3 Hash Function Implementation

A hardware implementation of the SHA-3 cryptographic hash function in Verilog

Project Overview

This project involved implementing the SHA-3 (Secure Hash Algorithm 3) cryptographic hash function in hardware using Verilog. SHA-3 is the latest member of the Secure Hash Algorithm family of standards, published by NIST in 2015. The implementation focused on achieving high throughput while maintaining low latency and efficient resource utilization.

Technical Details

  • Implemented the Keccak-f[1600] permutation function, the core of SHA-3
  • Designed a pipelined architecture for high throughput
  • Optimized for FPGA implementation with efficient resource usage
  • Implemented all five SHA-3 variants (SHA3-224, SHA3-256, SHA3-384, SHA3-512, and SHAKE)
  • Achieved a throughput of 1.6 Gbps on a Xilinx Artix-7 FPGA

Key Features

  • Configurable hash length support
  • Parallel processing capabilities
  • Low latency design
  • Resource-efficient implementation
  • Comprehensive test suite for verification

Implementation Details

The implementation uses a 5-stage pipeline to process the Keccak-f[1600] permutation rounds. Each stage handles a specific part of the algorithm:

  1. Input stage: Message padding and state initialization
  2. θ stage: Theta function implementation
  3. ρ and π stages: Rho and Pi functions combined
  4. χ stage: Chi function implementation
  5. ι stage: Iota function and output generation

Results

The implementation was successfully synthesized and tested on a Xilinx Artix-7 FPGA. Key performance metrics:

  • Maximum clock frequency: 200 MHz
  • Throughput: 1.6 Gbps
  • Latency: 24 clock cycles per block
  • Resource utilization: 12,000 LUTs, 6,000 FFs
