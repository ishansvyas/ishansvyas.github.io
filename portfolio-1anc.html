Ancestry Estimator

Using DNA data to trace your ancestry

Project Overview

This project combined bioinformatics, genetics, and computer programming to analyze ancestral yDNA and trace patterns of human migration. The goal was to develop a custom software tool that could map raw genetic data to geographic regions over time. This was the first programming project I ever did, so I learned a lot but it doesn't work very robustly.

Some Helpful Definitions

  • Y-DNA: The DNA passed down from fathers to sons. This is the DNA that is used for the ancestry estimator.
  • mtDNA: The DNA passed down from mothers to daughters. Autosomal DNA (DNA that isn't mtDNA or Y-DNA) recombines randomly in the population, so it is not useful for ancestry estimation.
  • SNPs: Single Nucleotide Polymorphisms, which defines the location at which a single base pair in DNA can vary.
  • Haplogroups: A group of people who share a common ancestor, and have a similar pattern of mutations in their Y-DNA.

Project Scope

The project was implemented in R. After an initial phase of learning the language and its data handling capabilities, I wrote a program that:

  • Imported genetic data and population reference databases.
  • Aligned and merged mutation data across multiple sources.
  • Identified relevant single nucleotide polymorphisms (SNPs) associated with known haplogroups.
  • Printed inferred ancestral paths and likely geographic origins.

Biological Context

The program focused on Y-DNA and mtDNA mutations, which are passed down paternally and maternally, respectively. By tracing these mutations and comparing them to published reference datasets, the software could generate rough maps of ancestral migration routes and origin points. This applied both historical population data and modern genomics to a real-world lineage analysis.
